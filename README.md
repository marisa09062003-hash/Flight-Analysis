# Flight-Analysis
 I launched this project to investigate how the subtle “habits” in pilot control during aircraft landings affect safety.  Using landing data generated by a simulator, we aim to detect early signs of human error and derive improvement strategies by comparing and analyzing the flight logs of an ideal landing with those of other flights. 
 
# Overview
This repository serves as the starting point for a project aimed at analyzing flight telemetry data to identify pilot-specific patterns and potential human errors during landing. By leveraging data generated from flight simulators, this project seeks to quantify "pilot habits" and derive actionable insights for safer flight performance.

# Project Background
The foundation of this analysis is a series of flight simulation logs. By comparing various flight attempts against an "ideal" landing (the 11th flight), we are beginning to uncover the quantitative differences between successful landings and those that deviate from the norm. This initial stage of analysis focuses on data cleaning, preprocessing, and calculating similarity scores to rank flight performance.

# Current Analysis (Flight_analysis_attenmpt1.ipynb)

Data Processing: Cleansing and structuring raw simulation telemetry (11.txt, etc.).

Comparative Analysis: Utilizing the 11th flight as a baseline to measure the similarity of subsequent flight attempts.

Visualization: Plotting key flight dynamics (such as Pitch) to track the stability and consistency of maneuvers.

Future Roadmap
The goal is to evolve this analysis into a more sophisticated Deep Learning model:

Anomaly Detection: Implementing LSTM/GRU models to detect potential human errors in real-time.

Pilot Profiling: Identifying and classifying individual pilot tendencies through their unique maneuvering "signature."

Performance Optimization: Developing personalized feedback systems to help pilots improve their landing techniques.
